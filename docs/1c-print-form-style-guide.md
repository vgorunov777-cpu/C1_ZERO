# Стайлгайд печатных форм 1С (MXL)

## Назначение

Правила разработки печатных форм (внешние обработки с макетом MXL) для AI-агентов и разработчиков.

---

## Макет (Template.xml / MXL DSL)

### Выравнивание текста

**Основные текстовые блоки** (преамбула, пункты договора, описательные ячейки) — горизонтальное выравнивание **По ширине** (`"align": "justify"`).

Исключения, где выравнивание другое:
- Заголовок документа — по центру (`"align": "center"`)
- Подписи — по центру (`"align": "center"`)
- Колонтитулы мелким шрифтом — по центру

```json
"styles": {
    "text-block": {
        "align": "justify",
        "valign": "top",
        "wrap": true
    }
}
```

### Именование областей

- Основная область документа — `Документ`
- Имена областей — по-русски, без пробелов

## Код заполнения (BSL)

### Заполнение параметров — только через структуру

Параметры области табличного документа заполнять **исключительно через `Область.Параметры.Заполнить(Структура)`**, а не напрямую через `Область.Параметры.ИмяПараметра = Значение`.

**Правильно:**
```bsl
ЗначенияПараметров = Новый Структура;
ЗначенияПараметров.Вставить("ДатаДокумента",  Формат(ТекДата, "ДФ='dd MMMM yyyy'") + " г.");
ЗначенияПараметров.Вставить("НомерДатаТД",    НомерДатаТД);
ЗначенияПараметров.Вставить("ФИОПолное",      ФИОПолное);
ОбластьДокумент.Параметры.Заполнить(ЗначенияПараметров);
```

**Неправильно:**
```bsl
ОбластьДокумент.Параметры.ДатаДокумента = Формат(ТекДата, "ДФ='dd MMMM yyyy'") + " г.";
ОбластьДокумент.Параметры.НомерДатаТД   = НомерДатаТД;
```

### Избегать двойных символов в параметрах

Если параметр уже содержит служебный символ (например, `НомерТД = "№" + ...`), не дублировать его в шаблоне:

```json
// Неправильно — будет "№№186/23"
"template": "Трудового договора от [ДатаТДСтрока] №[НомерТД]."

// Правильно
"template": "Трудового договора от [ДатаТДСтрока] [НомерТД]."
```

## Получение данных из конфигурации ЗУП/ERP

### Данные организации

Не читать реквизиты справочника `Организации` напрямую — использовать готовые функции конфигурации.

**ИНН, КПП, полное наименование** — через `РегламентированнаяОтчетностьВызовСервера.ПолучитьСведенияОбОрганизации`:

```bsl
УстановитьПривилегированныйРежим(Истина);
Сведения = Новый СписокЗначений;
Сведения.Добавить("", "НаимЮЛПол");   // полное наименование
Сведения.Добавить("", "ИННЮЛ");        // ИНН
Сведения.Добавить("", "КППЮЛ");        // КПП
Сведения.Добавить("", "ОГРН");         // ОГРН
Сведения.Добавить("", "ТелОрганизации");
ОргСведения = РегламентированнаяОтчетностьВызовСервера.ПолучитьСведенияОбОрганизации(
    Организация, ДатаДокумента, Сведения);
УстановитьПривилегированныйРежим(Ложь);

НаименованиеПолное = ОргСведения.НаимЮЛПол;
ИНН  = ОргСведения.ИННЮЛ;
КПП  = ОргСведения.КППЮЛ;
ОГРН = ОргСведения.ОГРН;
```

**Юридический и фактический адрес** — через `УправлениеКонтактнойИнформациейЗарплатаКадры`:

```bsl
// Шаг 1: загрузить адреса всех организаций одним вызовом
МассивОрганизаций = Новый Массив;
МассивОрганизаций.Добавить(Организация);
АдресаОрганизаций = УправлениеКонтактнойИнформациейЗарплатаКадры.АдресаОрганизаций(МассивОрганизаций);

// Шаг 2: получить конкретный адрес
ОписаниеЮрАдреса = УправлениеКонтактнойИнформациейЗарплатаКадры.АдресОрганизации(
    АдресаОрганизаций,
    Организация,
    Справочники.ВидыКонтактнойИнформации.ЮрАдресОрганизации);
АдресЮридический = ОписаниеЮрАдреса.Представление;

// Для фактического адреса:
// Справочники.ВидыКонтактнойИнформации.ФактАдресОрганизации
```

> Источник: `ОбщийМодуль.УправлениеКонтактнойИнформациейЗарплатаКадры`
> Источник: `ОбщийМодуль.РегламентированнаяОтчетностьВызовСервера`

### Данные организации — торговые сценарии (ERP/УТ)

Для торговых документов (реализация, поступление, счёт) реквизиты организации и контрагента можно получить через `ОрганизацииСервер` и `ФормированиеПечатныхФорм`.

**Выборочные поля организации** через `ОрганизацииСервер.СведенияОбОрганизации`:

```bsl
// Поля задаются строкой через запятую или пустой строкой — тогда все
// Возвращает структуру с полями: Наименование, НаименованиеПолное, ИНН, КПП, ОГРН,
//   ЮридическийАдрес, ФактическийАдрес, Телефон и т.д.
СведенияОрг = ОрганизацииСервер.СведенияОбОрганизации(Организация, "ИНН,КПП,НаименованиеПолное", ДатаДокумента);
ИНН = СведенияОрг.ИНН;
КПП = СведенияОрг.КПП;
```

**Банковский счёт организации** через `ОрганизацииСервер.БанковскийСчетОрганизации`:

```bsl
СведенияБанка = ОрганизацииСервер.БанковскийСчетОрганизации(Организация, "НомерСчета,Банк,БИК,КоррСчет", ДатаДокумента);
```

> Источник: `ОбщийМодуль.ОрганизацииСервер` — `СведенияОбОрганизации`, `ДополнительныеСведенияОрганизации`, `БанковскийСчетОрганизации`

### Реквизиты контрагента (торговые документы)

**Универсально для юр. и физ. лица** (организация или контрагент) — через `ФормированиеПечатныхФорм.СведенияОЮрФизЛице`:

```bsl
// ЮрФизЛицо — ссылка на Организацию или Контрагента
// Возвращает структуру: Наименование, НаименованиеПолное, ИНН, КПП, ОГРН,
//   АдресЮридический, АдресФактический, Телефон, ФИО (для ФЛ), БанковскиеРеквизиты и т.д.
СведенияСтороны = ФормированиеПечатныхФорм.СведенияОЮрФизЛице(
    Контрагент,         // ссылка
    ДатаДокумента,      // на дату
    Истина,             // ДляФизЛицаТолькоИнициалы
    БанковскийСчет);    // Неопределено — без банка
```

Или вариант с заполнением в существующую структуру:

```bsl
Сведения = Новый Структура;
ФормированиеПечатныхФорм.ЗаполнитьСведенияОЮрФизЛице(
    Сведения, Контрагент, ДатаДокумента);
```

**Реквизиты контрагента** (ИНН, КПП, юр. адрес) через `ПартнерыИКонтрагенты`:

```bsl
// ДатаСведений — Неопределено означает «актуальные»
РеквизитыКА = ПартнерыИКонтрагенты.РеквизитыКонтрагента(Контрагент, ДатаДокумента);
// Поля: ИНН, КПП, НаименованиеПолное, ...
```

> Источник: `ОбщийМодуль.ФормированиеПечатныхФорм` — `СведенияОЮрФизЛице`, `ЗаполнитьСведенияОЮрФизЛице`
> Источник: `ОбщийМодуль.ПартнерыИКонтрагенты` — `РеквизитыКонтрагента`

### Адрес и телефон по объекту (универсальные хелперы)

Для любого объекта с контактной информацией (организация, контрагент, физ. лицо):

```bsl
// Простые хелперы — только для печати, возвращают строку
АдресЮр = ФормированиеПечатныхФорм.ПолучитьАдресИзКонтактнойИнформации(
    Контрагент, "ЮридическийАдрес", ДатаДокумента);

Телефон = ФормированиеПечатныхФорм.ПолучитьТелефонИзКонтактнойИнформации(Контрагент);

Email   = ФормированиеПечатныхФорм.ПолучитьЭлектроннуюПочтуИзКонтактнойИнформации(Контрагент);
```

Если нужен **произвольный вид** или работа с несколькими объектами сразу — `УправлениеКонтактнойИнформацией`:

```bsl
// Возвращает строку представления (адрес, телефон и т.п.)
АдресСтрока = УправлениеКонтактнойИнформацией.ПредставлениеКонтактнойИнформацииОбъекта(
    Контрагент,
    Справочники.ВидыКонтактнойИнформации.ЮридическийАдресКонтрагента,
    ", ",           // разделитель
    ДатаДокумента);
```

> Источник: `ОбщийМодуль.ФормированиеПечатныхФорм` — `ПолучитьАдресИзКонтактнойИнформации`, `ПолучитьТелефонИзКонтактнойИнформации`, `ПолучитьЭлектроннуюПочтуИзКонтактнойИнформации`
> Источник: `ОбщийМодуль.УправлениеКонтактнойИнформацией` — `ПредставлениеКонтактнойИнформацииОбъекта`

### Подписи (ответственные лица организации)

Для области «Подписи» в макете — должность и ФИО со стороны организации.

```bsl
// Получить одно ответственное лицо по виду (Руководитель, ГлавныйБухгалтер и т.д.)
ДанныеРук = ОтветственныеЛицаСервер.ПолучитьДанныеОтветственногоЛица(
    Организация,
    ДатаДокумента,
    Перечисления.ОтветственныеЛицаОрганизаций.Руководитель);
// Поля результата: ФизическоеЛицо, Должность, Наименование, ДолжностьСсылка, ...

ДолжностьРук = ДанныеРук.Должность;     // строка
ФИОРук       = ДанныеРук.Наименование;  // строка ФИО
```

Если нужны **все** ответственные лица сразу — одним вызовом:

```bsl
// Возвращает структуру с ключами по именам значений перечисления
// ОтветственныеЛицаОрганизаций: Руководитель, ГлавныйБухгалтер и т.д.
// Для каждого ключа: %Вид%, %Вид%Должность, %Вид%ДолжностьСсылка, %Вид%Наименование, %Вид%Ссылка
ОтвЛица = ОтветственныеЛицаСервер.ПолучитьОтветственныеЛицаОрганизации(
    Организация, ДатаДокумента);

ФИОБухгалтера       = ОтвЛица.ГлавныйБухгалтер;           // СправочникСсылка.ФизическиеЛица
ДолжностьБухгалтера = ОтвЛица.ГлавныйБухгалтерДолжность;  // строка
```

> Источник: `ОбщийМодуль.ОтветственныеЛицаСервер` — `ПолучитьДанныеОтветственногоЛица`, `ПолучитьОтветственныеЛицаОрганизации`

### Данные сотрудника — ФИО, документ, адрес прописки

Используется механизм временных таблиц `КадровыйУчет`. Сначала нужна ВТ с парами `(Сотрудник, Период)`.

**Доступные поля** строки `КадровыеДанные`:
`ФИОПолные`, `ФамилияИО`, `АдресПоПропискеПредставление`, `ДокументПредставление`, `Пол`, `Страна`, `КоличествоДнейОтпускаОбщее`, `КлассУсловийТруда`, `EMailПредставление`

```bsl
// Предполагается что уже есть МенеджерВременныхТаблиц и ВТ с именем "ВТСотрудникиПериоды"
// с полями Сотрудник, Период

ОписательВременныхТаблиц = КадровыйУчет.ОписательВременныхТаблицДляСоздатьВТКадровыеДанныеСотрудников(
    Запрос.МенеджерВременныхТаблиц,
    "ВТСотрудникиПериоды");           // имя вашей ВТ с отборами

КадровыйУчет.СоздатьВТКадровыеДанныеСотрудников(
    ОписательВременныхТаблиц,
    Истина,                            // ТолькоРазрешенные
    "ФИОПолные,ФамилияИО,АдресПоПропискеПредставление,ДокументПредставление");

// Затем в запросе используем ВТКадровыеДанныеСотрудников:
// ВТКадровыеДанныеСотрудников.ФИОПолные            — "Иванов Иван Иванович"
// ВТКадровыеДанныеСотрудников.ФамилияИО            — "Иванов И. И."
// ВТКадровыеДанныеСотрудников.АдресПоПропискеПредставление — адрес регистрации
// ВТКадровыеДанныеСотрудников.ДокументПредставление — паспортные данные строкой
```

> Источник: `ОбщийМодуль.КадровыйУчет` — `СоздатьВТКадровыеДанныеСотрудников`, `ОписательВременныхТаблицДляСоздатьВТКадровыеДанныеСотрудников`

### ФИО краткое (расшифровка подписи)

```bsl
// Принимает строку "Иванов Иван Иванович" или структуру с полями Фамилия/Имя/Отчество
// Возвращает "И. И. Иванов"
ФИОКраткое = ФизическиеЛицаЗарплатаКадры.РасшифровкаПодписи(ФИОПолные);
```

> Источник: `ОбщийМодуль.ФизическиеЛицаЗарплатаКадры` — `РасшифровкаПодписи(ФИОПолные)`

Если `ФизическиеЛицаЗарплатаКадры` недоступен (УТ / ERP без кадрового контура), используй `ФизическиеЛицаУТ`:

```bsl
// Полное ФИО по ссылке физ. лица на дату
ФИОПолное   = ФизическиеЛицаУТ.ФамилияИмяОтчество(ФизическоеЛицо, ДатаДокумента);

// Инициалы + фамилия: "Иванов И. И."
ФИОКраткое  = ФизическиеЛицаУТ.ФамилияИнициалыФизЛица(ФизическоеЛицо, ДатаДокумента);
```

> Источник: `ОбщийМодуль.ФизическиеЛицаУТ` — `ФамилияИмяОтчество`, `ФамилияИнициалыФизЛица`

### Паспортные данные (развёрнуто)

`ДокументПредставление` из `КадровыйУчет` — готовая строка. Если нужны отдельные поля (серия, номер, кем выдан):

```bsl
// РегистрСведений.ДокументыФизическихЛиц.СрезПоследних
Запрос.Текст =
"ВЫБРАТЬ ПЕРВЫЕ 1
|   Д.Серия, Д.Номер, Д.ДатаВыдачи, Д.КемВыдан, Д.КодПодразделения
|ИЗ
|   РегистрСведений.ДокументыФизическихЛиц.СрезПоследних(
|       &ТекущаяДата, Физлицо = &ФизическоеЛицо) КАК Д
|ГДЕ
|   Д.ВидДокумента.Наименование ПОДОБНО ""%паспорт%""";
```

### Адрес регистрации физического лица (альтернатива)

Если `КадровыйУчет` недоступен (внешняя обработка без ВТ), можно получить напрямую:

```bsl
Запрос.Текст =
"ВЫБРАТЬ
|   КИ.Представление КАК Адрес
|ИЗ
|   Справочник.ФизическиеЛица.КонтактнаяИнформация КАК КИ
|ГДЕ
|   КИ.Ссылка = &ФизическоеЛицо
|   И КИ.Вид = ЗНАЧЕНИЕ(Справочник.ВидыКонтактнойИнформации.АдресПоПропискеФизическиеЛица)";
```

### Логотип и факсимиле организации

```bsl
// Вывести логотип организации в область МакетаОбласть
// ИмяОбласти — имя области макета, куда вставляется картинка
ФормированиеПечатныхФорм.ВывестиЛоготипВТабличныйДокумент(
    Макет, ОбластьШапки, "ЛоготипОрганизации", Организация);

// Вывести факсимиле (подпись) руководителя
// ПараметрыПечати — Неопределено или структура с доп. настройками
ФормированиеПечатныхФорм.ВывестиФаксимилеВТабличныйДокумент(
    Макет, ОбластьПодписей, Организация, Неопределено);
```

> Источник: `ОбщийМодуль.ФормированиеПечатныхФорм` — `ВывестиЛоготипВТабличныйДокумент`, `ВывестиФаксимилеВТабличныйДокумент`
