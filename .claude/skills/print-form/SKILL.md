# /print-form — Маршрутизатор печатных форм

Точка входа для любой работы с макетами табличных документов 1С (Template.xml / MXL). Направляет к нужному скиллу вместо ручного парсинга XML.

## Таблица маршрутизации

| Задача | Скилл | Пример |
|--------|-------|--------|
| Понять структуру макета (области, параметры) | `/mxl-info` | «Какие области и параметры в макете?» |
| Получить JSON из Template.xml для анализа/правки | `/mxl-decompile` | «Покажи макет в читаемом виде» |
| Создать макет с нуля по описанию | `/mxl-compile` | «Создай макет накладной М-11» |
| Создать макет по изображению/скану | `/img-grid` → `/mxl-compile` | «Вот скан формы, сделай макет» |
| Конвертировать XLSX в Template.xml | `/xlsx-to-mxl` | «Вот Excel-файл, сделай из него макет» |
| Модифицировать существующий макет | `/mxl-decompile` → правка → `/mxl-compile` | «Добавь колонку в макет» |
| Проверить макет после изменений | `/mxl-validate` | «Проверь макет на ошибки» |
| Добавить макет к обработке/объекту | `/template-add` | «Добавь макет ПФ_MXL_Отчет к обработке» |
| Написать BSL-код заполнения макета | `/mxl-info` → код | «Напиши процедуру печати» |

## Антипаттерны — что НЕЛЬЗЯ делать

- **НЕ читай Template.xml через Read tool** — это тысячи строк XML с индексными ссылками, нечитаемых без разрешения. Используй `/mxl-info` или `/mxl-decompile`.
- **НЕ парсь XML руками через Grep/Bash/скрипты** — все парсеры уже написаны и протестированы.
- **НЕ пиши собственные Python/PowerShell-парсеры для Template.xml** — используй готовые скиллы.
- **НЕ редактируй Template.xml через Edit tool** напрямую — индексы палитр и форматов взаимозависимы, любая ручная правка может сломать макет. Используй цикл decompile → правка JSON → compile.

## Рабочие циклы

### Анализ макета
```
/mxl-info <путь к Template.xml>
/mxl-info <ProcessorName> <TemplateName>
```
С текстовым содержимым: добавь `-WithText`

### Создание с нуля
1. Прочитать спецификацию: `docs-1c-skills/mxl-dsl-spec.md`
2. Написать JSON-определение макета
3. `/mxl-compile` → Template.xml
4. `/mxl-validate` → проверка
5. `/mxl-info` → верификация

### Создание по изображению
1. `/img-grid` → сетка для определения пропорций колонок
2. Написать JSON с `"Nx"` ширинами и `"page": "A4-landscape"`
3. `/mxl-compile` → `/mxl-validate` → `/mxl-info`

### Конвертация из XLSX
1. `/xlsx-to-mxl` → Template.xml
2. `/mxl-decompile` → JSON (для добавления областей/параметров)
3. Правка JSON → `/mxl-compile` → `/mxl-validate`

### Модификация существующего
1. `/mxl-decompile` → JSON
2. Правка JSON
3. `/mxl-compile` → `/mxl-validate`

### Полный цикл печатной формы (EPF)
1. `/epf-init` → обработка
2. `/template-add` → макет SpreadsheetDocument
3. `/mxl-compile` → содержимое макета
4. Код модуля объекта (процедура печати)
5. `/epf-bsp-init` → регистрация БСП
6. `/epf-bsp-add-command` → команда печати
7. `/epf-build` → сборка

## Канонический гайд

Полная документация: **`docs-1c-skills/mxl-guide.md`** — прочитать через Read tool при первом использовании.

Спецификация JSON DSL: **`docs-1c-skills/mxl-dsl-spec.md`** — прочитать перед написанием JSON-определения макета.
